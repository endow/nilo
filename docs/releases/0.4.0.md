# Release 0.4.0

Suggested tag: `v0.4.0`

## リリースノート（日本語）

Nilo 0.4.0 は、roadmap 出力の安全性と roadmap workflow の案内を整える minor release です。生成ファイルの出力先を誤って既存文書に向けた場合に上書きを拒否し、roadmap の相談、proposal 作成、import/adopt、human acceptance、task-plan までの流れを CLI help と README でより正確に示します。

### 変更点

- `nilo roadmap discuss`、`assess`、`summary`、`task-plan`、`export` が既存の非生成 Markdown を誤って上書きしないようにしました。
- `nilo roadmap adopt` が human roadmap 出力先の安全性を state 変更前に確認するようにしました。
- roadmap 系コマンドの `--file` に加えて `--output` alias を追加し、生成物の出力先指定を読みやすくしました。
- README / README.en と `nilo help ai` の roadmap 手順を、discussion context から RoadmapProposal の import/adopt、人間承認、task-plan へ進む実際の流れに合わせました。
- AGENTS 向け runtime instruction の roadmap 記述を簡潔にし、重複した human gate 文言を整理しました。
- roadmap 出力安全性、`--output` alias、AI 向け案内の回帰テストを追加しました。

### 検証

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは既存 DB を破壊する schema 変更はありません。

## Release Notes (English)

Nilo 0.4.0 is a minor release that improves roadmap output safety and roadmap workflow guidance. Generated roadmap commands now refuse to overwrite unrelated existing Markdown files, and the CLI help plus READMEs describe the real flow from roadmap discussion context to RoadmapProposal import/adopt, human acceptance, and task planning.

### Changes

- Made `nilo roadmap discuss`, `assess`, `summary`, `task-plan`, and `export` refuse accidental overwrites of existing non-generated Markdown files.
- Made `nilo roadmap adopt` validate the human roadmap output path before changing roadmap state.
- Added `--output` as an alias for `--file` on roadmap commands that write generated artifacts.
- Updated README / README.en and `nilo help ai` to match the actual roadmap flow: discussion context, RoadmapProposal import/adopt, human acceptance, then task planning.
- Simplified roadmap wording in AGENTS runtime instructions and removed duplicated human-gate phrasing.
- Added regression coverage for roadmap output safety, the `--output` alias, and AI-facing guidance.

### Verification

- `nilo check --task task_d23f425e5aae "env PYTHONPATH=src python tests/run_shards.py --all --jobs auto" --project nilo --mode full --timeout 600`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not include a destructive DB schema change.
