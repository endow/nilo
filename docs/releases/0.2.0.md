# Release 0.2.0

Suggested tag: `v0.2.0`

## リリースノート（日本語）

Nilo 0.2.0 では、リリース前のバージョン判断、roadmap-first の作業誘導、MCP の対象 repository 確認を強化しました。AI エージェントが複数 workspace や大きめの変更を扱うときに、対象の取り違えや roadmap 抜きの実装へ進みにくくするための minor release です。

### 変更点

- release recipe で `target_version` が未指定の場合、現在バージョンと最新 git tag から patch / minor 候補を提示するようにしました。
- CLI 変更、recipe 変更、roadmap / review / failure log など主要 workflow の変更がある場合は minor version を推奨するようにしました。
- `nilo help ai` と README に release recipe のバージョン提案ルールを追加しました。
- 大きな作業で roadmap を先に通すべき場合の `status --ai` / `next` の誘導を強化しました。
- Nilo MCP に repository identity guard と multi-workspace routing を追加し、`project_root` / `workspace` で対象 repository を明示できるようにしました。
- MCP の identity mismatch 時は通常 payload を返さず、CLI fallback で確認する運用を runtime instructions に追記しました。
- `nilo workspace resolve` を追加し、workspace 引数から repository root と DB path を確認できるようにしました。

### 検証

- `python -m unittest discover tests`
- `nilo check "python -m unittest discover tests" --project nilo --timeout 300`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは DB schema の変更はありません。

## Release Notes (English)

Nilo 0.2.0 strengthens release version advice, roadmap-first work routing, and repository identity checks for MCP workflows. This minor release is intended to reduce target mix-ups and direct implementation of larger changes without roadmap context when AI agents work across multiple workspaces or workflow-level changes.

### Changes

- The release recipe now suggests patch and minor candidates from the current version and latest git tag when `target_version` is omitted.
- Minor versions are recommended for CLI changes, recipe changes, roadmap / review / failure log workflow changes, and other major user-facing workflow updates.
- Added the release recipe version advice rules to `nilo help ai` and the README files.
- Strengthened `status --ai` / `next` guidance when large work should go through roadmap first.
- Added repository identity guard and multi-workspace routing to the Nilo MCP server, with explicit `project_root` / `workspace` targeting.
- Updated runtime instructions so MCP identity mismatches return no normal payload and agents fall back to CLI checks.
- Added `nilo workspace resolve` to inspect the repository root and DB path resolved from a workspace argument.

### Verification

- `python -m unittest discover tests`
- `nilo check "python -m unittest discover tests" --project nilo --timeout 300`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not change the DB schema.
