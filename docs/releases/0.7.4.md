# v0.7.4

## リリースノート（日本語）

Nilo 0.7.4は、unborn branchで開始したタスクの完了報告検証と、release workflowの状態遷移を修正するpatch releaseです。

### 変更点

- unborn repositoryで開始したタスクにGit empty treeを明示的な比較基準として記録し、初回コミットをタスク差分として検証できるようにしました。
- 旧形式の`base_commit=null`タスクについて、開始snapshotまたは安全なローカルverification evidenceからunborn開始を判定し、既存タスクにも後方互換を提供しました。
- 検証済みdirty treeと初回コミットの内容が同一ならevidenceをcurrentとして維持し、検証後の内容変更はstaleとして検出します。
- タスク開始前から存在したuntracked fileを内容ハッシュで区別し、変更されていないファイルをタスク差分へ混入させないようにしました。
- release commit後のclean treeで成功済みfull checkを`required_full_check`へ昇格し、公開承認ゲートへ進めない状態遷移を修正しました。
- release taskをrejectまたはdeferした際、関連recipe runをcancelし、保留中の公開操作を消去するようにしました。
- Ruff lint baselineの解消、extensionless changed file pathの処理、commit後のverification evidence維持など、0.7.3以降の品質修正を含みます。

### 検証

- full check: `PYTHONPATH=src python tests/run_shards.py --all --jobs auto`、exit_code=0
- full snapshot verification: `verification_648ac457ee71`
- release changed check: exit_code=0

### Upgrade / Install Notes

- pipxのeditable installでは、更新したworking treeがそのまま反映されます。
- 通常インストールでは`pipx upgrade nilo`または利用中のパッケージ管理手順で0.7.4へ更新してください。
- DB schemaは起動時migrationで`base_snapshot`を追加します。手動migrationは不要です。

## Release Notes (English)

Nilo 0.7.4 is a patch release that fixes completion-report validation for tasks started on unborn branches and corrects release workflow state transitions.

### Changes

- Records Git's empty tree as the explicit comparison base for tasks started in unborn repositories, allowing initial commits to be validated as task changes.
- Adds backward compatibility for legacy tasks with `base_commit=null` by inferring an unborn start from the saved start snapshot or safe local verification evidence.
- Keeps evidence current when a verified dirty tree is committed unchanged and marks it stale when content changes after verification.
- Distinguishes pre-existing untracked files by content hash so unchanged files are not mixed into task changes.
- Promotes a successful post-commit full check to `required_full_check` on a clean tree, allowing the release workflow to reach its public approval gate.
- Cancels related recipe runs and clears pending public operations when a release task is rejected or deferred.
- Includes quality fixes since 0.7.3, including removal of the Ruff lint baseline, extensionless changed-file handling, and preservation of verification evidence across commits.

### Verification

- Full check: `PYTHONPATH=src python tests/run_shards.py --all --jobs auto`, exit_code=0
- Full snapshot verification: `verification_648ac457ee71`
- Release changed check: exit_code=0
- Recipe doctor: exit_code=0
- Change range: `v0.7.2..4aa6f1d`

### Upgrade / Install Notes

- Editable pipx installs pick up the updated working tree directly.
- For regular installations, run `pipx upgrade nilo` or use the package-management procedure appropriate for your environment to install 0.7.4.
- The startup migration adds `base_snapshot` automatically; no manual database migration is required.
