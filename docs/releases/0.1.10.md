# Release 0.1.10

Suggested tag: `v0.1.10`

## リリースノート（日本語）

Nilo 0.1.10 では、人間が読む通常出力と AI 向け状態表示を日本語中心に整えました。内部状態値、DB、JSON 出力は英語の安定した識別子のまま維持し、外部連携との互換性を保っています。

### 変更点

- `status`, `next`, `task show`, `evidence show`, `review show`, `failure list/summary/show`, `doctor` 系の人間向け表示を日本語ラベル中心にしました。
- `--ai` 出力も日本語を基本にし、必要な状態値は `未提出 (missing)` のように内部値を括弧で併記するようにしました。
- 日本語表示ラベルを `src/nilo/display_labels.py` に集約し、状態、重大度、分類、フィールド名、真偽値の表示変換を共通化しました。
- `--json` 出力と DB 内部値は英語キー・英語 enum のまま保つテストを追加しました。
- README に、通常出力は日本語中心、`--json` は外部連携向けに日本語化しないという表示言語方針を追記しました。
- README の利用説明を整理し、0.1.9 リリースノートのアップグレード説明を補正しました。

### 検証

- `nilo check "python -m unittest discover tests" --task task_d9e201a95b82 --timeout 300`

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは DB schema の変更はありません。

## Release Notes (English)

Nilo 0.1.10 makes human-readable command output and AI-oriented status views Japanese-first while preserving stable English identifiers in internal state, the database, and JSON output.

### Changes

- Localized human-facing labels for `status`, `next`, `task show`, `evidence show`, `review show`, `failure list/summary/show`, and selected `doctor` surfaces.
- Made `--ai` output Japanese-first while keeping machine enum values in parentheses where useful, such as `未提出 (missing)`.
- Centralized display label conversion in `src/nilo/display_labels.py` for statuses, severities, categories, field names, and booleans.
- Added tests that verify `--json` output and DB enum values remain English and machine-readable.
- Documented the display-language policy in README: normal output is Japanese-first, while `--json` remains unchanged for integrations.
- Streamlined README usage text and corrected the 0.1.9 upgrade note.

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not change the DB schema.
