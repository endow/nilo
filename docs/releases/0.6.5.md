# v0.6.5

## リリースノート（日本語）

Nilo 0.6.5 は、リリースレシピと work entrypoint の扱いを堅くする patch release です。空のリリースノートテンプレートを公開しない保護、release evidence の記録と再利用、active task 中の work recipe 選択、review finding 対応後の証跡を改善しました。

### 変更点

- 空のリリースノートテンプレートを GitHub release として公開しないようにしました。
- `nilo work` が active task を安全に扱い、release recipe task adoption の文脈を壊さないようにしました。
- release evidence review findings への対応として、検証証跡、release metadata、write-path coverage を補強しました。
- release publish 前の full check と release note handoff の保護を改善しました。

### 検証

- release recipe は公開操作を人間承認ゲートとして扱います。
- 公開前に `PYTHONPATH=src python tests/run_shards.py --all --jobs auto` の full check を実行する必要があります。
- 公開は人間承認後に tag / push / GitHub release を実行します。

### 根拠

- 変更範囲: `v0.6.4..HEAD`
- 対象コミット: `cf8431e`, `aa79673`, `79f16da`, `21ddf6b`
- target_version: `0.6.5`
- バージョン判断: 現在バージョン `0.6.4` と最新タグ `v0.6.4` が一致しているため、release recipe のローカル規約に従い次 patch の `0.6.5` を採用。
- version advisor は変更範囲の広さから minor candidate `0.7.0` も提示しましたが、今回の公開準備では明示 target として `0.6.5` を記録します。

## Release Notes (English)

Nilo 0.6.5 is a patch release that hardens the release recipe and the work entrypoint. It prevents empty release-note templates from being published, improves release evidence recording and reuse, protects work recipe selection while a task is active, and tightens evidence after review-finding fixes.

### Changes

- Prevented empty release-note templates from being published as GitHub releases.
- Made `nilo work` handle active tasks safely without breaking release recipe task adoption context.
- Strengthened verification evidence, release metadata, and write-path coverage after release evidence review findings.
- Improved protection around the full check before release publish and release-note handoff.

### Verification

- The release recipe treats public operations as a human approval gate.
- `PYTHONPATH=src python tests/run_shards.py --all --jobs auto` must pass before publishing.
- Tag, push, and GitHub release creation remain gated by explicit human approval.

### Evidence

- Change range: `v0.6.4..HEAD`
- Commits: `cf8431e`, `aa79673`, `79f16da`, `21ddf6b`
- target_version: `0.6.5`
- Version decision: current version `0.6.4` matches latest tag `v0.6.4`, so the release recipe local rule selects the next patch version, `0.6.5`.
- The version advisor also suggested minor candidate `0.7.0` because of the breadth of changes; this release handoff records `0.6.5` as the explicit target.
