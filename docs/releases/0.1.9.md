# Release 0.1.9

Suggested tag: `v0.1.9`

## リリースノート（日本語）

Nilo 0.1.9 では、`failure_logs` を失敗台帳として扱うための CLI と表示機能を追加しました。

### 変更点

- `nilo failure list`, `summary`, `show`, `resolve`, `ignore` を追加しました。
- `failure_logs` に `source`, `actor`, `related_id`, `snapshot`, `status`, `resolved_at`, `resolved_by`, `resolution_note` を追加しました。
- report import、human outcome reject/rework、verification の secret detection 由来の失敗記録に発生元と状態を保存するようにしました。
- `task show --ai` に対象 task の open failure を最大 5 件表示し、failure logs は必須ルールではないことを明示しました。
- `status --ai` と `doctor ai-context` に compact な failure summary を追加しました。
- legacy learning tables を復活させず、failure logs を instruction 生成や completion gate に使わない境界を維持しました。
- review finding が解決済みでも status が `review_changes_requested` に残る状態投影を修正しました。

### 検証

- `python -m pytest -q`
- `nilo check "python -m pytest -q" --project nilo --task task_0103e31bd3b8 --timeout 300`
- `nilo recipe doctor --project nilo`

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。既存 DB は additive migration で `failure_logs` の新カラムが追加されます。

## Release Notes (English)

Nilo 0.1.9 adds Failure Ledger support for `failure_logs`, making recorded failures easier to inspect, summarize, resolve, and ignore without turning them into automatic rules.

### Changes

- Added `nilo failure list`, `summary`, `show`, `resolve`, and `ignore`.
- Extended `failure_logs` with `source`, `actor`, `related_id`, `snapshot`, `status`, `resolved_at`, `resolved_by`, and `resolution_note`.
- Recorded source and status metadata for failures from report import, human outcome rejection/rework, and verification secret detection.
- Added up to five open task failure observations to `task show --ai` with an explicit note that failure logs are not mandatory rules.
- Added compact failure summaries to `status --ai` and `doctor ai-context`.
- Preserved the boundary that legacy learning tables are not recreated and failure logs are not injected into instructions or completion gates.
- Fixed task status projection so resolved review findings no longer leave a task stuck in `review_changes_requested`.

### Verification

- `python -m pytest -q`
- `nilo check "python -m pytest -q" --project nilo --task task_0103e31bd3b8 --timeout 300`
- `nilo recipe doctor --project nilo`

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. Existing databases receive the new `failure_logs` columns through additive migrations.
