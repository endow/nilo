# v0.7.7

## リリースノート（日本語）

Nilo 0.7.7 は、レビュー実行基盤、Task 完了判定、roadmap 証跡集約、release recipe の一貫性を改善する patch release です。

### 変更点

- Claude / Codex / Grok adapter を共通 registry と coordinator に統合し、レビュー試行の状態、エラー分類、fallback policy、MCP lease / reaper を整備しました。
- Claude レビューはユーザーによる明示要求がある場合だけ実行し、未解決レビュー指摘がある Task の完了を防ぐ安全柵を追加しました。
- roadmap の検証証跡を集約し、置き換え済みの pending revision を無視するよう修正しました。
- 検証済み commit 後も完了判定を維持し、独立した work Task の routing と明示的な work intent の扱いを改善しました。
- recipe 推定で参照パスと更新対象を区別し、release recipe が単一 Task を継続利用するようにしました。

## Release Notes (English)

Nilo 0.7.7 is a patch release improving review execution, task completion guards, roadmap evidence aggregation, and release recipe consistency.

### Changes

- Unified the Claude, Codex, and Grok adapters behind a shared registry and coordinator, with review-attempt state, error classification, fallback policy, and MCP lease/reaper support.
- Restricted Claude reviews to explicit user requests and prevented task completion while actionable review findings remain unresolved.
- Aggregated roadmap verification evidence and ignored superseded pending revisions.
- Preserved completion validity after verified commits and improved independent work-task routing and explicit work-intent handling.
- Distinguished reference paths from update targets during recipe inference and kept release recipes within a single task.
