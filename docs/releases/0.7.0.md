# v0.7.0

## リリースノート（日本語）

Nilo 0.7.0 は、AI 完了報告の取り込み前検証を強化するリリースです。完了報告 markdown を DB に取り込む前に形式と変更ファイル一覧を検査できるようにし、release や通常作業の完了証跡をより確実に残せるようにしました。

### 変更点

- `nilo report validate` を追加し、完了報告 markdown の形式、必須セクション、`changed_files` の内容を import 前に確認できるようにしました。
- `nilo instruct` の完了報告手順に validate preflight を組み込み、AI が report import 前に差分と untracked files を確認する流れを明示しました。
- 完了報告 import 周辺の検証を追加し、空欄や不正な変更ファイル一覧を早めに検出できるようにしました。

## Release Notes (English)

Nilo 0.7.0 strengthens preflight validation for AI completion reports. Completion report markdown can now be checked before it is imported into the DB, making release and normal-work evidence easier to verify.

### Changes

- Added `nilo report validate` to check completion report markdown structure, required sections, and `changed_files` before import.
- Updated `nilo instruct` completion-report guidance so AI agents validate reports and inspect tracked and untracked changes before importing.
- Added coverage around completion report import validation to catch blank sections and invalid changed-file lists earlier.
