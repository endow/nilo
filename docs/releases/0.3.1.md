# Release 0.3.1

Suggested tag: `v0.3.1`

## リリースノート（日本語）

Nilo 0.3.1 は、Task 完了記録と作業キュー監査の一貫性を改善する patch release です。状態遷移を共通レイヤーに集約し、完了記録の証跡・人間確認・監査イベントをより厳密に扱うことで、AI 作業の完了判断と後続確認を追跡しやすくしました。

### 変更点

- Task、Todo、roadmap、review、failure の主要な状態変更を `transitions` レイヤーに集約し、遷移イベントを記録するようにしました。
- `nilo task complete` / `nilo done` の完了記録で、AI と人間の判断、検証証跡、人間確認メモ、予約付き完了をより明確に扱うようにしました。
- 完了記録の監査を追加し、疑わしい人間完了、古い snapshot、証跡不足などを `nilo doctor completions` と `nilo queue --audit` で確認できるようにしました。
- `nilo queue` が通常表示では完了監査の詳細を混ぜず、必要な場合に `--audit` で invalid completion を確認できるようにしました。
- AI context、MCP review import、failure resolution、roadmap / todo 操作が共通 transition guard を使うように更新しました。
- completion audit と transition layer の回帰テストを追加しました。

### 検証

- `nilo check "python -m unittest discover tests" --project nilo --mode full --timeout 300`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは既存 DB を破壊する schema 変更はありません。

## Release Notes (English)

Nilo 0.3.1 is a patch release that improves consistency around Task completion records and work-queue audits. State changes now flow through a shared transition layer, making completion evidence, human confirmation, and audit events easier to trace after AI-assisted work.

### Changes

- Centralized major Task, Todo, roadmap, review, and failure state changes in the `transitions` layer with recorded transition events.
- Clarified completion recording in `nilo task complete` and `nilo done`, including AI versus human decisions, verification evidence, human decision notes, and completion with reservations.
- Added completion auditing for suspicious human completions, stale snapshots, and missing evidence through `nilo doctor completions` and `nilo queue --audit`.
- Kept normal `nilo queue` output focused while exposing invalid completion details through the explicit `--audit` option.
- Updated AI context, MCP review import, failure resolution, and roadmap / todo operations to use shared transition guards.
- Added regression tests for completion audit behavior and the transition layer.

### Verification

- `nilo check "python -m unittest discover tests" --project nilo --mode full --timeout 300`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not include a destructive DB schema change.
