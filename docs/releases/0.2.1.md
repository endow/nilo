# Release 0.2.1

Suggested tag: `v0.2.1`

## リリースノート（日本語）

Nilo 0.2.1 では、AI エージェントが許可された repository 境界の外へ書き込まないための project boundary write fence と、レビュー handoff を MCP 経由で扱いやすくする既定公開を追加しました。複数 workspace や自己変更を含む運用で、対象 repository と書き込み範囲をより明確に扱うための patch release です。

### 変更点

- project boundary resolver を追加し、現在の repository root、git root、Nilo DB、書き込み可能 scope、自己変更可否を AI 向け文脈に含めるようにしました。
- `AGENTS.md` の Nilo managed block に project boundary と forbidden path 情報を出力するようにしました。
- CLI handler の write fence を追加し、許可された repository 外へのレポート、roadmap、review handoff、quality handoff などの出力を拒否するようにしました。
- MCP の review handoff tools を既定で公開し、review workflow で必要な handoff 操作を MCP から使いやすくしました。
- project boundary と MCP review handoff のテストを追加しました。

### 検証

- `python -m unittest discover tests`
- `nilo check "python -m unittest discover tests" --project nilo --timeout 300`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは DB schema の変更はありません。

## Release Notes (English)

Nilo 0.2.1 adds a project boundary write fence so AI agents avoid writing outside the allowed repository boundary, and exposes review handoff MCP tools by default. This patch release makes target repositories and writable scope clearer for multi-workspace and self-modification workflows.

### Changes

- Added a project boundary resolver that reports the current repository root, git root, Nilo DB, writable scope, and self-modification status in AI-facing context.
- Added project boundary and forbidden path details to the Nilo managed block in `AGENTS.md`.
- Added CLI handler write fences that reject report, roadmap, review handoff, and quality handoff outputs outside the allowed repository.
- Exposed MCP review handoff tools by default so review workflow handoff operations are available through MCP.
- Added tests for project boundary handling and MCP review handoff exposure.

### Verification

- `python -m unittest discover tests`
- `nilo check "python -m unittest discover tests" --project nilo --timeout 300`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not change the DB schema.
