# Release 0.2.2

Suggested tag: `v0.2.2`

## リリースノート（日本語）

Nilo 0.2.2 では、AI 向けの次作業案内、roadmap 承認待ちの表示、reviewer dispatch の Windows 対応を改善しました。大きめの作業を roadmap に寄せる判断をより実用的にし、release や広範囲変更以外で full test を既定案内しすぎないようにした patch release です。

### 変更点

- `nilo status --ai` / `nilo next` / `nilo help ai` の案内を整理し、Task 化、Todo、roadmap-first の使い分けをより明確にしました。
- roadmap 承認待ちの UX を改善し、内部用語だけで止まらず、人間が判断しやすい作業計画の説明を促すようにしました。
- roadmap 自動誘導のしきい値を調整し、複数ファイルに触れるだけの明確な bug fix は通常 task として進めやすくしました。
- focused implementation recipe と AI 向け文脈で、release や広範囲・高リスク変更以外は full test ではなく targeted / focused verification を優先する案内にしました。
- roadmap workflow を ID-aware にし、roadmap discussion / task plan / execution の対象取り違えを減らしました。
- Windows 環境で reviewer dispatch の実行ファイル解決と repository guard を修正しました。
- verification mode と active task start guardrail のテストを追加し、関連する README / design documentation を更新しました。

### 検証

- `python -m unittest discover tests`
- `nilo check "python -m unittest discover tests" --project nilo --mode full --timeout 300`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは DB schema の変更はありません。

## Release Notes (English)

Nilo 0.2.2 improves AI-facing next-step guidance, pending roadmap approval output, and Windows reviewer dispatch handling. This patch release makes roadmap routing more practical and avoids over-recommending full tests outside release, broad-risk, or high-risk work.

### Changes

- Clarified `nilo status --ai`, `nilo next`, and `nilo help ai` guidance for choosing between Task creation, Todo capture, and roadmap-first work.
- Improved the pending roadmap approval UX so agents explain the work plan in human-reviewable terms instead of stopping at internal terminology.
- Adjusted the roadmap auto-routing threshold so coherent bug fixes can proceed as normal tasks even when they touch multiple files.
- Updated the focused implementation recipe and AI-facing context to prefer targeted or focused verification unless the work is release, broad-risk, or high-risk.
- Made roadmap workflow handling ID-aware to reduce target mix-ups across roadmap discussion, task planning, and execution.
- Fixed Windows reviewer dispatch executable resolution and repository guard behavior.
- Added tests for verification modes and the active task start guardrail, with related README and design documentation updates.

### Verification

- `python -m unittest discover tests`
- `nilo check "python -m unittest discover tests" --project nilo --mode full --timeout 300`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not change the DB schema.
