# Release 0.5.0

Suggested tag: `v0.5.0`

## リリースノート（日本語）

Nilo 0.5.0 は、リリース作業を最後まで進めやすくする minor release です。明示承認後に tag 作成、main push、tag push、GitHub release 作成、完了記録まで Nilo が実行できるようにし、通常の status 表示では高速 snapshot を使って大きな作業ツリーでも待ち時間を抑えます。

### 変更点

- `nilo recipe approve-public --execute` を追加し、承認済みの release public operations を Nilo が実行して recipe completion まで記録できるようにしました。
- release recipe の `next` / AI context / project summary に、承認後に実行する `--execute` 付きコマンドを表示するようにしました。
- 公開操作の実行前に clean tree を確認し、失敗時に public gate を消費して再実行不能にならないようにしました。
- release recipe の説明を、手動 handoff ではなく approval-gated execution を案内する内容に更新しました。
- status / AI context 系の表示で fast git snapshot を使えるようにし、詳細確認が必要な場合は full snapshot を使う形にしました。
- 非 git 環境の smoke task で verification evidence が誤って `recorded` 扱いになる回帰を修正しました。
- `nilo doctor performance` を追加し、git snapshot の diff hash 計算時間や対象 path 数を確認できるようにしました。
- release public-operation 実行、dirty tree 失敗時の再試行性、fast snapshot の回帰テストを追加しました。

### 検証

- `nilo check --task task_eb9c0f043189 "sh -c 'PYTHONPATH=src python tests/run_shards.py --all --jobs auto'" --project nilo --mode full --timeout 900`
- `nilo recipe doctor --project nilo`
- `nilo check --task task_eb9c0f043189 "sh -c 'NILO_NO_UPDATE_CHECK=1 PYTHONPATH=src python -m nilo status --project nilo'" --project nilo --mode targeted --timeout 60`

`PYTHONPATH=src python -m unittest discover tests` was also attempted through `nilo check`, but timed out at both 300 and 600 seconds. The full shard runner above completed successfully and is the recorded full-suite evidence for this release.

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは既存 DB を破壊する schema 変更はありません。

## Release Notes (English)

Nilo 0.5.0 is a minor release that makes release work easier to carry all the way through. After explicit approval, Nilo can now create the tag, push `main`, push the tag, create the GitHub release, and record release completion. Normal status surfaces can also use fast git snapshots so large working trees spend less time waiting on exact diff hashes.

### Changes

- Added `nilo recipe approve-public --execute` so Nilo can execute approved release public operations and record recipe completion.
- Made release recipe `next`, AI context, and project summary surfaces show the approval-gated `--execute` command.
- Added a clean-tree guard before public operation execution and kept the public gate retryable when execution fails before crossing it.
- Updated release recipe wording to guide approval-gated execution instead of manual handoff.
- Added fast git snapshot support for status and AI context surfaces, with full snapshots still available when exact freshness checks are needed.
- Fixed a regression where smoke tasks outside a git repository could incorrectly report verification evidence as `recorded`.
- Added `nilo doctor performance` for inspecting git snapshot diff-hash timing and path counts.
- Added regression coverage for release public-operation execution, dirty-tree retry behavior, and fast snapshots.

### Verification

- `nilo check --task task_eb9c0f043189 "sh -c 'PYTHONPATH=src python tests/run_shards.py --all --jobs auto'" --project nilo --mode full --timeout 900`
- `nilo recipe doctor --project nilo`
- `nilo check --task task_eb9c0f043189 "sh -c 'NILO_NO_UPDATE_CHECK=1 PYTHONPATH=src python -m nilo status --project nilo'" --project nilo --mode targeted --timeout 60`

`PYTHONPATH=src python -m unittest discover tests` was also attempted through `nilo check`, but timed out at both 300 and 600 seconds. The full shard runner above completed successfully and is the recorded full-suite evidence for this release.

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not include a destructive DB schema change.

## GitHub release execution

After explicit approval, use Nilo's approval-gated execution path:

```bash
nilo recipe approve-public --project nilo --approval 'v0.5.0 を tag/push/release して' --execute
```

Equivalent human-gated commands:

```bash
git tag v0.5.0
git push origin main
git push origin v0.5.0
gh release create v0.5.0 --title "v0.5.0" --notes-file docs/releases/0.5.0.md
```
