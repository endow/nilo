# Release 0.3.0

Suggested tag: `v0.3.0`

## リリースノート（日本語）

Nilo 0.3.0 では、AI 作業開始時の状態確認、review handoff、roadmap-first 判断、残件確認の体験をまとめて改善しました。`nilo next --project` を軽量化し、新しい `nilo queue` で未完了 Task と対応待ち Todo を素早く確認できるようにした minor release です。

### 変更点

- `nilo check` が複数 active task を検出した場合に、検証証跡の紐づけ先を推測せず `--task` 指定を求める理由と、無関係な task に記録しない判断を明確にしました。
- `nilo status --ai` / `task status --ai` の completion 表示を `現在タスク完了診断` に変更し、完了操作の結果ではなく現在 task の完了条件診断であることを分かりやすくしました。
- generated runtime instruction に、`nilo review status --task <task_id> --format json` の正しい例と、`review status` に `--project` を付けない注意を追加しました。
- active task がある場合の `nilo next --project` を軽量化し、project summary 全体を構築せず先頭 action を返すようにしました。
- 新しい `nilo queue` コマンドを追加し、未完了 Task と `open` / `ready` / `triaged` / `blocked` / `requires_roadmap` / `deferred` Todo を human / JSON 出力で確認できるようにしました。
- roadmap planning の承認後タスク、review handoff instruction、queue command の回帰テストを追加しました。

### 検証

- `nilo check "python -m unittest discover tests" --project nilo --mode full --timeout 900`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは DB schema の変更はありません。

## Release Notes (English)

Nilo 0.3.0 improves AI startup status checks, review handoff guidance, roadmap-first routing, and quick work-queue inspection. This minor release makes `nilo next --project` lighter and adds `nilo queue` for quickly checking unfinished Tasks and actionable Todos.

### Changes

- Clarified the `nilo check` error for multiple active tasks so agents understand that verification evidence must attach to exactly one task and should not be recorded on an unrelated task.
- Renamed AI-facing completion readiness output to `現在タスク完了診断`, making it clearer that the line is a current-task completion-condition diagnosis rather than the result of a completion attempt.
- Added generated runtime instruction guidance with the correct `nilo review status --task <task_id> --format json` example and a warning not to pass `--project` to `review status`.
- Sped up `nilo next --project` when an active task exists by avoiding full project summary construction before printing the first action.
- Added `nilo queue` with human and JSON output for unfinished Tasks and actionable Todos in `open`, `ready`, `triaged`, `blocked`, `requires_roadmap`, and `deferred` states.
- Added regression tests for roadmap-approved work, review handoff instructions, and queue output.

### Verification

- `nilo check "python -m unittest discover tests" --project nilo --mode full --timeout 900`
- `nilo recipe doctor --project nilo`
- `nilo status --project nilo`

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not change the DB schema.
