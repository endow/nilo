# Release 0.3.2

Suggested tag: `v0.3.2`

## リリースノート（日本語）

Nilo 0.3.2 は、release workflow の安全柵、状態遷移の堅牢化、AI 向け作業入口、並列 test shard 実行を整える patch release です。公開操作を人間ゲートに閉じ込めつつ、Task / review / workflow の状態記録と監査をより追跡しやすくしました。

### 変更点

- release recipe の workflow context を追加し、release 中は無関係な `next` へ流れず、tag / push / GitHub release などの公開操作を明示承認まで保留するようにしました。
- Task / workflow / review 周辺の状態遷移、write path allowlist、snapshot、state audit を強化し、直接書き込みや競合状態を検出しやすくしました。
- Nilo の AI 向け entry command と status / next 出力を整理し、作業開始時の現在地確認と release recipe 継続を軽くしました。
- `nilo test` と `tests/run_shards.py` に並列 shard runner を追加し、変更領域や失敗 shard に応じた検証を走らせやすくしました。
- reviewer command lookup でアクセス不能な `PATH` entry を扱えるようにし、review dispatch 周辺の失敗を減らしました。
- release workflow、transition、state audit、write path、shard runner の回帰テストを追加しました。

### 検証

- `nilo check --task task_1b219430b8cb "env PYTHONPATH=src python tests/run_shards.py --all --jobs auto" --project nilo --mode full --timeout 300`
- `nilo check --task task_1b219430b8cb "nilo recipe doctor --project nilo" --project nilo --mode targeted --timeout 120`
- `nilo status --ai --project nilo`

### アップグレード / インストール

新規インストール:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

git checkout からインストール済みの場合は、作業ツリーが clean な状態で次を実行してください。

```bash
nilo upgrade
```

`nilo upgrade` は `git pull --ff-only`、editable reinstall、migration を実行します。既存 DB がある場合は migration 前に `.nilo/backups/` へ `before-upgrade` バックアップを作成します。このリリースでは既存 DB を破壊する schema 変更はありません。

## Release Notes (English)

Nilo 0.3.2 is a patch release that tightens release workflow guardrails, hardens state transitions, improves AI-facing entry points, and adds parallel test shard execution. Public operations stay behind a human gate while Task, review, and workflow state records become easier to audit.

### Changes

- Added release recipe workflow context so active releases do not drift into unrelated `next` work, and public operations such as tag, push, and GitHub release creation remain pending until explicit approval.
- Hardened Task, workflow, and review state transitions, write path allowlists, snapshots, and state audits so direct writes and concurrency issues are easier to detect.
- Simplified Nilo's AI-facing entry commands and status / next output for lighter startup checks and release recipe continuation.
- Added `nilo test` and `tests/run_shards.py` parallel shard running to make focused and failure-based verification easier.
- Handled inaccessible `PATH` entries during reviewer command lookup to reduce review dispatch failures.
- Added regression coverage for release workflow, transitions, state audit, write paths, and the shard runner.

### Verification

- `nilo check --task task_1b219430b8cb "env PYTHONPATH=src python tests/run_shards.py --all --jobs auto" --project nilo --mode full --timeout 300`
- `nilo check --task task_1b219430b8cb "nilo recipe doctor --project nilo" --project nilo --mode targeted --timeout 120`
- `nilo status --ai --project nilo`

### Upgrade / Install

Fresh install:

```bash
git clone https://github.com/endow/nilo.git
cd nilo
python -m pip install -e .
```

If Nilo was installed from a git checkout, update from a clean working tree with:

```bash
nilo upgrade
```

`nilo upgrade` runs `git pull --ff-only`, reinstalls the editable package, and runs migrations. If an existing DB is present, Nilo creates a `before-upgrade` backup under `.nilo/backups/` before migration. This release does not include a destructive DB schema change.
