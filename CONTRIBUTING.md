# Contributing

Nilo は現在、experimental な個人 / 小規模チーム向けツールです。bug fix、ドキュメント修正、小さな usability 改善、既存挙動のテスト追加は歓迎します。

task model、evidence model、MCP behavior、DB schema などの大きな変更は、実装前に issue で相談してください。

Nilo は convenience shortcut より、evidence、reproducibility、human-readable state を優先します。変更提案では、その方針を崩さないかを明確にしてください。
